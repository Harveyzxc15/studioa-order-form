一、系統概覽

這套系統解決什麼問題？

北一區六間門市向採購提出商品需求時，常忘記 CC Bob，導致無法追蹤下訂時間與到貨狀態。

本系統分為兩個部分：

元件用途使用者位置
📋 需求表單 填寫商品資料，自動產生格式正確的信件 門市人員 GitHub Pages（任何裝置可開）
📦 追蹤後台 追蹤所有訂購需求的下單、出貨、到貨狀態 Bob 本機 localhost:5556

整體流程圖

1

門市填寫表單

開啟表單網頁，選擇門市、填入商品資料

2

產生信件並發送

系統自動帶入收件人（Ellen / Clare）、副本（門市 + 店長 + Bob）、主旨、內文

✉️ 信件從門市自己的 Mail 發出，Bob 的信箱自動收到副本
3

後台自動掃描（每天 09:00）

系統自動從 Bob 的 Apple Mail 抓取訂購信，建立追蹤記錄

4

Bob 確認已下單

Clare / Ellen 回覆後，Bob 在後台點「確認已下單」並輸入實際下單日期，開始計算等待天數

5

EPB 自動偵測到貨

系統查詢 EPB 調撥記錄（090J 出貨中 / 140eb 已到貨），自動更新狀態

6

逾期 Discord 通知

超過 14 個工作天未到貨，自動發送 Discord 通知

🏪

二、門市操作流程

門市人員
⚠️
寄出前請店長確認信件內容所有資料正確無誤，信件一旦發出即視為正式訂購需求。

Step 1 — 開啟表單

使用任何裝置（電腦、手機、平板）開啟：

📋 開啟表單

Step 2 — 選擇門市

在「基本資訊」選擇正確的門市名稱。選錯門市會導致副本收件人錯誤，採購回覆也可能漏通知。

Step 3 — 新增商品

點「＋ 新增商品」，選擇商品類型後填入欄位。
同一封信有多筆需求時，重複新增即可，不需分開寄送。

ℹ️
存貨代碼必須為 8 碼數字（例：07312305）。格式錯誤會立即提示，無法產生信件。

Step 4 — 確認預覽

點「📧 產生信件預覽」，仔細核對收件人、副本、主旨與信件內文，確認無誤再繼續。

⚠️
若發現資料有誤，點「← 返回修改」修正後重新產生預覽。

Step 5 — 開啟 Mail 發送

點「✉️ 開啟 Mail 發送」，Apple Mail 會自動帶入所有資訊。

在 Mail 中：

  1. 再次確認收件人、副本、主旨
  2. 若為 CTO 或 iPad 高容量，附上所需附件
  3. 確認無誤後按發送
✅
信件發出後不需要其他動作。Bob 的信箱會自動收到副本，系統每天早上 09:00 自動掃描並建立追蹤記錄。
📦

三、商品類型填寫說明

門市人員

🛒 原廠商品

適用一般庫存商品（非客製、非高容量 iPad）。

欄位說明
存貨代碼 *8 碼數字，從系統查詢（例：07312305）
商品名稱 *完整商品名稱（例：ME2J4TA/A Powerbeats Fit Jet Black）
數量 *需求數量
單價 *售價（例：6990）
是否已收訂金 *已全額付清 / 已收部分訂金 / 未結清
備註選填，其他補充說明

📱 iPad 高容量

適用：iPad Pro 512G/1TB/2TB、iPad Air 512G/1TB、CEL 版 iPad。

欄位說明
存貨代碼 *8 碼數字
商品名稱 *完整規格名稱（例：iPad Pro 13吋 M4 1TB Wi-Fi 黑色）
數量 *需求數量
是否已收訂金 *已全額付清 / 已收部分訂金 / 未結清
備註選填
📎
必附附件：客戶已結帳單據（含發票及訂購單）

⚙️ CTO 客製機

適用 Apple 官網客製化規格商品。需先在 Apple 官網設定好規格。

欄位說明
原存貨代碼 *原始機型的 8 碼存貨代碼（例：07312164）
原產品名稱 *原始機型名稱（例：MAC STUDIO/M4 Max/16C CPU/40C GPU/64GB/512G）
原價 *原始售價
客製規格 *客製內容（例：記憶體 128GB、硬碟 2TB）
客製後價格 *客製後售價
客製網址 *在 Apple 官網設定完成後複製的網址
備註選填
📎
必附附件：①客製機截圖 ②客戶已結帳單據（含發票及訂購單）

四、訂購追蹤後台操作

Bob

開啟後台

點選選單列 📊 → 📦 訂購需求追蹤，或直接開啟：

🖥️ 開啟後台

若頁面無法開啟，表示 Flask server 尚未啟動。點選選單列圖示會自動啟動。

後台功能說明

按鈕功能何時使用
🔄 掃描信件 手動掃描 Apple Mail，匯入新訂購信及採購回覆 想立即更新，不等每天 09:00
📡 EPB 查詢 查詢所有未到貨訂單的出貨與到貨狀態 想確認最新到貨狀態
✅ 確認已下單 標記採購已確認下單，輸入實際下單日期 收到 Clare / Ellen 回覆確認後
手動到貨 手動標記已到貨（EPB 查不到時） CTO 客製機 或 EPB 未更新
取消確認 取消已下單狀態 誤按時使用
刪除 刪除錯誤記錄 系統誤判匯入的記錄

確認已下單 — 日期輸入說明

點「確認已下單」後會彈出視窗要求輸入實際下單日期。

ℹ️
等待天數從這個日期起算，請輸入 Clare / Ellen 回覆確認當天的日期（格式：2026-05-28），而非今天。若當天即時確認，直接按 Enter 使用預設值即可。
📊

五、狀態流程與等待天數

Bob

狀態流程

⏳ 待採購確認 → 手動點確認 ✅ 已下單 → EPB 090J 🚚 出貨中 → EPB 140eb 🏪 已到貨

等待天數計算規則

項目說明
起算點Bob 點「確認已下單」時輸入的實際下單日期
終點EPB 偵測到 140eb（已到貨），否則顯示至今天
顯示顏色綠色 < 7 天　橘色 7–13 天　紅色 ≥ 14 天
已到貨顯示「共等 N 天」（灰色），不再計時

Discord 逾期通知

超過 7 個工作天（不含週六日）未到貨，系統自動發送 Discord 通知。

ℹ️
每筆訂單只通知一次，不會重複提醒。CTO 客製機因無法 EPB 自動偵測，需手動點「手動到貨」才會停止計時。

EPB 查詢說明

交易代碼意義對應狀態
090J調撥出（倉庫出貨，尚未到門市）🚚 出貨中
140eb調撥收（門市收貨入庫完成）🏪 已到貨

CTO 客製機無存貨代碼，EPB 無法自動偵測，需手動更新。

cd ~/order_tracker git add . && git commit -m "說明改了什麼" && git push # 重啟 server 讓修改生效 lsof -ti:5556 | xargs kill -9 python3 app.py &

自動掃描排程

launchd 每天 09:00 自動執行 auto_scan.py，完成以下工作：

  1. 掃描 Apple Mail，匯入新訂購信
  2. 掃描 Clare / Ellen 的採購回覆
  3. 檢查是否有超過 14 個工作天的未到貨訂單，送出 Discord 通知
# 手動執行（測試用） cd ~/order_tracker && python3 auto_scan.py # 查看執行 log cat /tmp/order_tracker_scan.log

常見問題

問題解法
後台無法開啟 點選單列 📊 → 📦 訂購需求追蹤，會自動重啟 server
掃描沒有匯入新信 確認信件主旨包含「原廠商品」「客製機需求」等關鍵字，且有 CC bob.yu@studioa.com.tw
EPB 查不到狀態 確認存貨代碼正確（8碼），CTO 請手動更新
表單頁面沒更新 按 ⌘+Option+R 強制重新整理
Discord 沒收到通知 確認 app.py 中 DISCORD_WEBHOOK 有填入，且訂單狀態為已下單但未到貨
